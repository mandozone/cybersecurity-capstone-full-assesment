# Cybersecurity Capstone – Full Security Assessment

## Overview
This project is a full-scope cybersecurity assessment conducted in a simulated enterprise environment (Artemis Lab). The engagement follows a structured 5-phase methodology aligned with real-world security operations and risk assessment practices.

The objective was to identify vulnerabilities, analyze potential threats, and provide actionable remediation strategies based on business impact.

---

## Assessment Methodology

### Phase 1: Reconnaissance
- Passive and active information gathering
- Identification of exposed assets and services

### Phase 2: Scanning & Enumeration
- Network and port scanning
- Service and version detection
- Initial attack surface mapping

### Phase 3: Vulnerability Identification
- Vulnerability scanning using industry tools
- Identification of misconfigurations and weaknesses

### Phase 4: Threat Assessment
- Risk analysis based on likelihood and impact
- Prioritization of vulnerabilities (CVSS-style thinking)

### Phase 5: Reporting
- Executive summary for stakeholders
- Technical report with detailed findings and remediation steps

---

## Tools & Technologies
- Nmap (Network Scanning)
- Nessus Essentials (Vulnerability Scanning)
- Wireshark (Traffic Analysis)
- Burp Suite (Web Application Testing)
- Kali Linux

---

## Key Findings
- Identified multiple exposed services increasing attack surface
- Detected misconfigured systems vulnerable to exploitation
- Observed weak security controls and potential entry points

---

## Business Impact
The identified vulnerabilities could allow unauthorized access, data exposure, and service disruption. If exploited, these weaknesses could lead to financial loss, reputational damage, and operational downtime.

---

## Remediation Recommendations
- Harden exposed services and close unnecessary ports
- Apply security patches and updates
- Implement network segmentation
- Strengthen access controls and monitoring
